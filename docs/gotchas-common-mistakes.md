# 常見誤解與陷阱清單 (Gotchas & Common Mistakes)

> 人類容易誤會弄錯的地方，整理成清單避免踩坑

---

## TL;DR

- **Cron 是鬧鐘，Heartbeat 是待辦清單** — 別搞混
- **Payload 是獨立指令來源** — 不會自動讀取 AGENTS.md/SOUL.md
- **sessionTarget 必須搭配正確的 payload.kind** — 混搭會導致不執行
- **一次性 cron 必須加 autoDelete** — 否則會留在清單中

---

## 概念混淆

### 1. Cron vs Heartbeat

**容易混淆原因：**
- 都跟「定時執行」有關
- 都在 workflow 中一起出現

**正確理解：**
- **Cron** = 觸發器（何時執行）
- **Heartbeat** = 任務內容（執行什麼）
- 關係：Cron 觸發 → 讀 HEARTBEAT.md → 執行任務

**類比：**
- Cron = 鬧鐘（定時響鈴）
- Heartbeat = 待辦清單（要做的事）

```
┌─────────────────────────────────────────────────┐
│  Cron 設定（鬧鐘）                                │
│  schedule: "*/30 * * * *"                       │
│  → 每 30 分鐘觸發一次                             │
└─────────────────┬───────────────────────────────┘
                  │
                  │ 觸發
                  ▼
┌─────────────────────────────────────────────────┐
│  讀取 HEARTBEAT.md（待辦清單）                    │
│  - 安全檢查                                      │
│  - 系統自我修復                                   │
│  - 主動驚喜                                      │
│  - 記憶維護                                      │
└─────────────────┬───────────────────────────────┘
                  │
                  │ 執行
                  ▼
┌─────────────────────────────────────────────────┐
│  執行結果                                        │
│  - 回覆 HEARTBEAT_OK（無事）                     │
│  - 或回報警告/驚喜                               │
└─────────────────────────────────────────────────┘
```

### 2. sessionTarget vs payload.kind

**容易混淆原因：**
- 兩者都影響執行方式
- 文件中沒有明確說明「必須配對」

**正確組合：**

```
┌──────────────────────────────────────────────────────────┐
│              正確組合 vs 錯誤組合                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ✅ 正確組合 1：輕量任務（<1 分鐘）                         │
│  ┌─────────────────────────────────────────┐            │
│  │  sessionTarget: main                    │            │
│  │  payload.kind: systemEvent              │            │
│  │  → 注入文字到主會話                      │            │
│  │  → 適合：提醒、心跳、輕量檢查             │            │
│  └─────────────────────────────────────────┘            │
│                                                          │
│  ✅ 正確組合 2：獨立任務（耗時、需隔離）                    │
│  ┌─────────────────────────────────────────┐            │
│  │  sessionTarget: isolated                │            │
│  │  payload.kind: agentTurn                │            │
│  │  → 啟動獨立會話                          │            │
│  │  → 適合：報告生成、API 呼叫、長任務       │            │
│  └─────────────────────────────────────────┘            │
│                                                          │
│  ❌ 錯誤組合（會導致不執行）                               │
│  ┌─────────────────────────────────────────┐            │
│  │  sessionTarget: main                    │            │
│  │  payload.kind: agentTurn                │ ← 錯！     │
│  └─────────────────────────────────────────┘            │
│  ┌─────────────────────────────────────────┐            │
│  │  sessionTarget: isolated                │            │
│  │  payload.kind: systemEvent              │ ← 錯！     │
│  └─────────────────────────────────────────┘            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**決策樹：**
```
需要定時自動跑任務？
├─ 任務 < 1 分鐘且需要主會話上下文？
│   └─ Yes → sessionTarget: main
│            payload.kind: systemEvent
└─ No
    └─ sessionTarget: isolated
        payload.kind: agentTurn
```

### 3. Payload 獨立性

**容易混淆原因：**
- 以為 isolated session 會自動繼承 AGENTS.md/SOUL.md
- 以為可以「依賴主會話的規則」

**事實：**
- Payload 是**獨立指令來源**
- **不會**自動讀取 AGENTS.md、SOUL.md
- 必須明確寫出所有需要的指令

**正確寫法：**

```
# ❌ 不好（假設會自動讀取 AGENTS.md）
幫我檢查系統狀態

# ✅ 正確（明確寫出所有指令）
Read HEARTBEAT.md if it exists (workspace context).
Follow it strictly.
Do not infer or repeat old tasks from prior chats.
If nothing needs attention, reply HEARTBEAT_OK.
```

**示意圖：**
```
┌──────────────────────────────────────────────────────────┐
│                    主會話（Main Session）                  │
│                                                          │
│  ✅ 可讀取：                                              │
│  - AGENTS.md（規則）                                      │
│  - SOUL.md（人格）                                        │
│  - USER.md（用戶資訊）                                    │
│  - 對話歷史                                              │
│                                                          │
└──────────────────────────────────────────────────────────┘
                         │
                         │ ❌ 不會自動繼承
                         ▼
┌──────────────────────────────────────────────────────────┐
│           Isolated Session（獨立會話）                    │
│                                                          │
│  ⚠️  Payload 是獨立指令來源                               │
│                                                          │
│  ❌ 不會自動讀取：                                        │
│  - AGENTS.md                                            │
│  - SOUL.md                                              │
│  - 主會話上下文                                          │
│                                                          │
│  ✅ 必須明確寫在 payload 中                               │
│                                                          │
│  範例：                                                  │
│  ┌──────────────────────────────────────┐               │
│  │  Read HEARTBEAT.md if it exists.     │               │
│  │  Follow it strictly.                 │               │
│  │  Do not infer or repeat old tasks.   │               │
│  │  If nothing needs attention,         │               │
│  │  reply HEARTBEAT_OK.                 │               │
│  └──────────────────────────────────────┘               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 實戰陷阱

### 4. 一次性 cron 忘記 autoDelete

**問題：**
- `schedule.kind: "at"`（一次性）
- 忘記加 `autoDelete: true`
- 過期的 cron job 會一直留在清單中

**解法：**
```yaml
schedule:
  kind: "at"
  at: "2026-03-07T09:00:00+08:00"
autoDelete: true  # ← 必須加
```

---

### 5. 規則變更後忘記同步 cron payload

**問題：**
- 改 AGENTS.md 的規則
- 以為 isolated session 會自動知道
- 但 payload 是獨立指令來源，不會自動繼承

**解法：**
每次改 AGENTS.md 的規則，檢查所有相關 cron 的 payload 是否需要同步更新。

---

### 6. Timeout 太短

**問題：**
- isolated session 有預設 timeout
- 外部 API 可能慢（幾秒到幾十秒）
- 執行腳本可能需要幾分鐘

**解法：**
```yaml
session:
  timeoutSeconds: 300  # 5 分鐘
```

## Debug 流程

### Cron 觸發了但沒執行？

```
Cron 觸發了但沒執行？
│
├─ 檢查 1：sessionTarget 與 payload.kind 是否匹配？
│   │
│   ├─ main → 必須是 systemEvent
│   └─ isolated → 必須是 agentTurn
│
├─ 檢查 2：Payload 是否明確？
│   │
│   └─ isolated session 不會自動讀取 AGENTS.md
│       → 必須在 payload 中明確寫出所有指令
│
├─ 檢查 3：Timeout 是否足夠？
│   │
│   └─ isolated session 有預設 timeout
│       → 外部 API 可能慢，需要預留時間
│
├─ 檢查 4：delivery.mode 是否正確？
│   │
│   ├─ announce → 自動投遞到指定頻道
│   └─ none → 由 session 內自行處理投遞
│
└─ 檢查 5：日誌中有錯誤訊息嗎？
    │
    └─ tail -100 /tmp/clawdbot/*.log | grep -i "error\|fail\|warn"
```
