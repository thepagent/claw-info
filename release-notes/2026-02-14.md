# OpenClaw v2026.02.14 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.2.14)

## 概述

本版本包含以下主要變化：

- **功能更新**
  - ⭐⭐⭐ Telegram: add poll sending
  - ⭐ Slack/Discord: add dmPolicy + allowFrom config aliases
  - ⭐⭐ Discord: allow exec approval prompts to target channels
  - ⭐ Sandbox: add sandbox.browser.binds
  - ⭐ Discord: add debug logging

- **安全性提升**
  - ⭐⭐⭐ Security/Memory-LanceDB: treat recalled memories as untrusted
  - ⭐⭐⭐ Security/Memory-LanceDB: require explicit autoCapture opt-in
  - ⭐⭐⭐ Security/BlueBubbles: require explicit mediaLocalRoots allowlists
  - ⭐⭐⭐ Security/Nostr: require loopback source
  - ⭐⭐⭐ Security/Signal: harden archive extraction
  - ⭐⭐⭐ Security/Windows: avoid shell invocation
  - ⭐⭐ Security/Agents: scope CLI process cleanup
  - ⭐⭐ Security/Agents: enforce workspace-root path bounds
  - ⭐⭐ Security/Agents (macOS): prevent shell injection
  - ⭐⭐ Security/Gateway: harden tool-supplied gatewayUrl overrides
  - ⭐⭐ Security/Gateway: block system.execApprovals.*
  - ⭐⭐ Security/Net: fix SSRF guard bypass
  - ⭐⭐ Security/Browser: harden browser control file upload/download
  - ⭐⭐ Security/Node Host: enforce rawCommand/argv consistency
  - ⭐⭐ Security/Exec approvals: prevent allowlist bypass
  - ⭐⭐ Security/Tlon: harden Urbit URL fetching
  - ⭐⭐ Security/Voice Call (Telnyx): require webhook signature
  - ⭐⭐ Security/Voice Call: require valid Twilio webhook signatures
  - ⭐⭐ Security/Discovery: stop treating Bonjour TXT as authoritative

- **錯誤修復**
  - ⭐⭐ CLI/Plugins: ensure exit after successful delivery
  - ⭐⭐ CLI/Plugins: run gateway_stop hooks
  - ⭐ WhatsApp: honor per-account dmPolicy overrides
  - ⭐ Telegram: exclude plugin commands when native is false
  - ⭐ LINE: return 200 OK for Verify requests
  - ⭐ Cron: deliver text-only output when delivery.to is set
  - ⭐ Cron/Slack: preserve agent identity
  - ⭐ Media: accept MEDIA:-prefixed paths
  - ⭐ Agents: deliver tool result media
  - ⭐ Agents/Image tool: allow workspace-local image paths
  - ⭐ Telegram: set webhook callback timeout handling
  - ⭐ Signal: preserve case-sensitive group IDs
  - ⭐ Feishu/Security: harden media URL fetching
  - ⭐ Security/Slack: compute command authorization for DM
  - ⭐ Security/iMessage: keep DM pairing-store identities out of group allowlist
  - ⭐ Security/Google Chat: deprecate users/ allowlists
  - ⭐ Security/Google Chat: reject ambiguous shared-path webhook routing
  - ⭐ Telegram/Security: require numeric Telegram sender IDs
  - ⭐ Telegram/Security: reject webhook startup when webhookSecret is missing

## 功能更新 (Features) - by Star Rating

### ⭐⭐⭐ Telegram: add poll sending ([#16209](https://github.com/openclaw/openclaw/pull/16209))
- **用途**：支援透過 openclaw message poll 發送投票
- **解決問題**：先前無法發送投票
- **影響**：提供投票功能支援（持續時間秒數、靜默投遞、匿名控制）

### ⭐ Slack/Discord: add dmPolicy + allowFrom config aliases
- **用途**：新增 dmPolicy + allowFrom 配置别名用於 DM 訪問控制
- **解決問題**：配置名稱不一致
- **影響**：簡化 DM 配置，保留舊配置支援

### ⭐⭐ Discord: allow exec approval prompts to target channels ([#16051](https://github.com/openclaw/openclaw/pull/16051))
- **用途**：允許 exec approval prompts 發送到頻道或 DM+頻道
- **解決問題**：exec approval 只能發送到 DM
- **影響**：提升 exec approval 的靈活性 ([#16051](https://github.com/openclaw/openclaw/pull/16051))

### ⭐ Sandbox: add sandbox.browser.binds ([#16230](https://github.com/openclaw/openclaw/pull/16230))
- **用途**：新增 sandbox.browser.binds 用於獨立配置瀏覽器容器的綁定掛載
- **解決問題**：無法獨立配置瀏覽器和執行容器的綁定掛載
- **影響**：提升 sandbox 配置的靈活性 ([#16230](https://github.com/openclaw/openclaw/pull/16230))

### ⭐ Discord: add debug logging for message routing decisions ([#16202](https://github.com/openclaw/openclaw/pull/16202))
- **用途**：為訊息路由決策添加 debug 日誌
- **解決問題**：調試時無法看到路由決策原因
- **影響**：改善 --debug 追蹤能力 ([#16202](https://github.com/openclaw/openclaw/pull/16202))

## 安全性提升 (Security) - by Star Rating

### ⭐⭐⭐ Security/Memory-LanceDB: treat recalled memories as untrusted ([#12524](https://github.com/openclaw/openclaw/pull/12524))
- **用途**：將召回的記憶視為不受信任的上下文
- **解決問題**：記憶中毒攻擊風險
- **影響**：防止記憶中毒，增加記憶處理的安全性

### ⭐⭐⭐ Security/Memory-LanceDB: require explicit autoCapture opt-in ([#12552](https://github.com/openclaw/openclaw/pull/12552))
- **用途**：需要明確的 autoCapture: true 選項（默認為禁用）
- **解決問題**：未經選擇自動捕獲 PII
- **影響**：防止未經意的 PII 捕獲

### ⭐⭐⭐ Security/BlueBubbles: require explicit mediaLocalRoots allowlists ([#16322](https://github.com/openclaw/openclaw/pull/16322))
- **用途**：要求明確的 mediaLocalRoots allowlists 用於本地媒體路徑讀取
- **解決問題**：本地文件洩漏風險
- **影響**：防止本地文件意外洩露 ([#16322](https://github.com/openclaw/openclaw/pull/16322))

### ⭐⭐⭐ Security/Nostr: require loopback source
- **用途**：要求 loopback 來源並阻止跨域配置文件變更/導入嘗試
- **解決問題**：跨域 API 調用
- **影響**：防止 Nostr 配置更改的未經授權訪問

### ⭐⭐⭐ Security/Signal: harden archive extraction during install
- **用途**：在安裝期間強化 signal-cli 歸檔解壓以防止路徑遍歷
- **解決問題**：歸檔解壓期間的路徑遍歷漏洞
- **影響**：防止安裝期間的任意文件寫入

### ⭐⭐⭐ Security/Windows: avoid shell invocation
- **用途**：生成子進程時避免調用 shell
- **解決問題**：cmd.exe 元字符注入
- **影響**：防止 Windows 上的命令注入攻擊

### ⭐⭐ Security/Agents: scope CLI process cleanup to owned child PIDs
- **用途**：將 CLI 進程清理限制到所屬的子進程 PID
- **解決問題**：共享主機上意外終止進程
- **影響**：防止殺死共享主機上的不相關進程

### ⭐⭐ Security/Agents: enforce workspace-root path bounds for apply_patch
- **用途**：在非沙箱模式下為 apply_patch 強制執行工作區根路徑邊界
- **解決問題**：目錄遍歷和符號鏈接逃逸寫入
- **影響**：防止工作區外的文件寫入

### ⭐⭐ Security/Agents (macOS): prevent shell injection
- **用途**：寫入 Claude CLI 金鑰鏈憑據時防止 shell 注入
- **解決問題**：不明確 CLI 參數的 shell 注入
- **影響**：防止 macOS 上的 Claude CLI 金鑰鏈注入漏洞 ([#15924](https://github.com/openclaw/openclaw/pull/15924))

### ⭐⭐ Security/Gateway: harden tool-supplied gatewayUrl overrides
- **用途**：限制工具提供的 gatewayUrl 覆蓋到 loopback 或配置的 gateway.remote.url
- **解決問題**：gatewayUrl 覆蓋的 SSRF 風險
- **影響**：防止 gatewayUrl 覆蓋的 SSRF 攻擊

### ⭐⭐ Security/Gateway: block system.execApprovals.* via node.invoke
- **用途**：阻止透過 node.invoke 的 system.execApprovals.*
- **解決問題**：執行批准的繞過
- **影響**：防止 exec approvals 的權限繞過

### ⭐⭐ Security/Net: fix SSRF guard bypass
- **用途**：修復透過完整形式 IPv4-mapped IPv6 文字的 SSRF 防護繞過
- **解決問題**：SSRF 防護被繞過
- **影響**：防止 SSRF 攻擊

### ⭐⭐ Security/Browser: harden browser control file upload/download
- **用途**：強化瀏覽器控制文件上傳/下載助手以防止路徑遍歷/本地文件洩露
- **解決問題**：瀏覽器控制的任意文件讀寫
- **影響**：防止瀏覽器控制的文件洩密

### ⭐⭐ Security/Node Host: enforce rawCommand/argv consistency
- **用途**：強制 system.run 的 rawCommand/argv 一致性
- **解決問題**：allowlist/approval 繞過
- **影響**：防止 system.exec 的權限繞過

### ⭐⭐ Security/Exec approvals: prevent allowlist bypass via shell expansion
- **用途**：防止 shell 扩展的 allowlist 繞過（主機 exec allowlist 模式）
- **解決問題**：allowlist 的 shell 扩展繞過
- **影響**：防止 exec approvals 的 allowlist 繞過

### ⭐⭐ Security/Tlon: harden Urbit URL fetching
- **用途**：強化 Urbit URL 抓取以防止 SSRF，通過默認阻止私人/內部主機
- **解決問題**：SSRF 攻擊
- **影響**：防止 Urbit 的 SSRF 攻擊

### ⭐⭐ Security/Voice Call (Telnyx): require webhook signature verification
- **用途**：接收入站事件時需要 webhook 簽名驗證
- **解決問題**：無效 webhook 請求
- **影響**：防止 webhook 請求偽造

### ⭐⭐ Security/Voice Call: require valid Twilio webhook signatures
- **用途**：即使啟用 ngrok 免費套餐 loopback 兼容模式，也需要有效的 Twilio webhook 簽名
- **解決問題**： Twilio webhook 的簽名驗證
- **影響**：防止 Twilio webhook 請求偽造

### ⭐⭐ Security/Discovery: stop treating Bonjour TXT records as authoritative
- **用途**：停止將 Bonjour TXT 記錄視為權威路由
- **解決問題**：服務發現的路由繞過
- **影響**：防止服務發現的路由欺骗

## 錯誤修復 (Bug Fixes) - by Star Rating

### ⭐⭐ CLI/Plugins: ensure exit after successful delivery ([#16491](https://github.com/openclaw/openclaw/pull/16491))
- **用途**：確保 openclaw message send 在成功交付後退出
- **解決問題**：一次性發送掛起
- **影響**：防止 CLI 發送命令掛起

### ⭐⭐ CLI/Plugins: run gateway_stop hooks ([#16580](https://github.com/openclaw/openclaw/pull/16580))
- **用途**：在 openclaw message 退出前運行註冊的 plugin gateway_stop hooks
- **解決問題**：plugin 資源清理不完整
- **影響**：確保 plugin 支持的渠道清理資源

### ⭐ WhatsApp: honor per-account dmPolicy overrides ([#10082](https://github.com/openclaw/openclaw/pull/10082))
- **用途**：遵循每帳戶 dmPolicy 覆蓋
- **解決問題**：帳戶級別設定未被遵守
- **影響**：確保帳戶級別 DM 策略正確執行

### ⭐ Telegram: exclude plugin commands when native is false ([#15132](https://github.com/openclaw/openclaw/pull/15132))
- **用途**：當 channels.telegram.commands.native 為 false 時，排除插件命令
- **解決問題**：插件命令與原生命令衝突
- **影響**：確保插件命令正確註冊

### ⭐ LINE: return 200 OK for Verify requests
- **用途**：對 Developers Console "Verify" 請求返回 200 OK
- **解決問題**：LINE webhook 前端驗證失敗
- **影響**：確保 LINE webhook 正確驗證

### ⭐ Cron: deliver text-only output when delivery.to is set
- **用途**：當 delivery.to 設置時直接傳遞純文本輸出
- **解決問題**：cron 收件人收到摘要而不是完整輸出
- **影響**：確保完整輸出傳遞到 cron 收件人

### ⭐ Cron/Slack: preserve agent identity ([#16242](https://github.com/openclaw/openclaw/pull/16242))
- **用途**：cron 作業傳遞輸出訊息時保留代理身份
- **解決問題**：cron 發送訊息時丢失身份
- **影響**：確保 cron 訊息保留代理名稱和圖標

### ⭐ Media: accept MEDIA:-prefixed paths ([#13107](https://github.com/openclaw/openclaw/pull/13107))
- **用途**：接受 MEDIA:-prefixed 路徑以防止 ENOENT
- **解決問題**：工具返回的本地媒體路徑被拒絕
- **影響**：支援工具返回的媒體路徑

### ⭐ Agents: deliver tool result media ([#11735](https://github.com/openclaw/openclaw/pull/11735))
- **用途**：Regardless of verbose level，向渠道傳遞工具結果媒體
- **解決問題**：工具結果媒體未傳遞
- **影響**：確保工具結果媒體正確傳遞

### ⭐ Agents/Image tool: allow workspace-local image paths
- **用途**：允許工作區本地 image paths
- **解決問題**：工作區圖像路徑被拒絕
- **影響**：支援工作區本地圖像路徑

### ⭐ Telegram: set webhook callback timeout handling
- **用途**：將 webhook callback 超時處理設定為 onTimeout: "return" (10s)
- **解決問題**：長時間處理導致 webhook 500s
- **影響**：防止 webhook 500s 和重試風暴

### ⭐ Signal: preserve case-sensitive group IDs ([#16748](https://github.com/openclaw/openclaw/pull/16748))
- **用途**：在规范化期间保留大小寫敏感的 group: target IDs
- **解決問題**：混合大小寫組 ID 失敗
- **影響**：確保 Signal 組 ID 正確處理

### ⭐ Feishu/Security: harden media URL fetching
- **用途**：強化 Feishu 媒體 URL 抓取以防止 SSRF 和本地文件洩露
- **解決問題**：SSRF 和本地文件洩露風險
- **影響**：防止 Feishu 的 SSRF 攻擊

### ⭐ Security/Slack: compute command authorization for DM
- **用途**：即使 dmPolicy=open，也計算 DM 斜線命令的命令授權
- **解決問題**：未授權用戶通過 DM 執行特權命令
- **影響**：防止未授權的 DM 命令執行

### ⭐ Security/iMessage: keep DM pairing-store identities out of group allowlist
- **用途**：DM 配對存儲身份不包含在群組 allowlist 授權中
- **解決問題**：跨上下文命令授權
- **影響**：防止 iMessage 的跨上下文授權

### ⭐ Security/Google Chat: deprecate users/ allowlists
- **用途**：棄用 users/ allowlists，將 users/... 視為不變的用戶 ID
- **解決問題**：allowlists 的歧義
- **影響**：簡化 Google Chat 的 allowlists

### ⭐ Security/Google Chat: reject ambiguous shared-path webhook routing
- **用途**：當多個 webhook 目標驗證成功時，拒絕含糊的共享路徑 webhook 路由
- **解決問題**：跨帳戶策略上下文錯誤路由
- **影響**：防止 Google Chat 的跨帳戶路由

### ⭐ Telegram/Security: require numeric Telegram sender IDs
- **用途**：對 allowlist 授權要求數字 Telegram sender IDs
- **解決問題**：@username principal 的 allowlist 錯誤
- **影響**：確保 Telegram 的正確 allowlist 授權

### ⭐ Telegram/Security: reject webhook startup when webhookSecret is missing
- **用途**：當 webhookSecret 缺失或為空時，拒絕 webhook 啟動
- **解決問題**：未驗證的 webhook 請求偽造
- **影響**：防止未驗證的 webhook 請求
