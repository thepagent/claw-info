# OpenClaw v2026.4.27 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.27)

## 概述

### 功能更新 (Features)
- ⭐⭐⭐ **Codex Computer Use 整合設定**：新增 `/codex computer-use status/install` 指令、marketplace 探索、自動安裝選項及 fail-closed MCP 檢查 ([#71842](https://github.com/openclaw/openclaw/pull/71842))
- ⭐⭐⭐ **DeepInfra 內建 Provider**：新增 DeepInfra 模型探索、圖片生成/編輯、TTS、影片生成、記憶嵌入，及 provider-owned 基礎 URL 策略（延續 [#53805](https://github.com/openclaw/openclaw/issues/53805)、[#48088](https://github.com/openclaw/openclaw/issues/48088) 等）
- ⭐⭐⭐ **騰訊 QQBot 完整群聊支援**：新增群組歷史追蹤、@提及閘控、串流回覆、統一媒體上傳及管線化架構重構 ([#70624](https://github.com/openclaw/openclaw/pull/70624))
- ⭐⭐⭐ **Plugin 啟動：manifest-first 元資料架構**：外掛啟動遷移至顯式 `activation.onStartup` 宣告，減少 Gateway 啟動工作量
- ⭐⭐ **騰訊 Yuanbao 頻道**：註冊 Yuanbao 外部頻道外掛、文檔入口及快速上手指南 ([#72756](https://github.com/openclaw/openclaw/pull/72756)、[#73443](https://github.com/openclaw/openclaw/pull/73443))
- ⭐⭐ **Plugin SDK 通用 Host Hooks**：新增 session 狀態、下一回合 context、信任工具策略、UI 描述子、事件排程器及執行期 plugin context 等 hooks ([#72287](https://github.com/openclaw/openclaw/pull/72287))
- ⭐⭐ **Plugin SDK 測試/合約重構**：大量測試 fixture、contract helper、channel 測試工具遷移至 focused SDK subpaths
- ⭐⭐ **Plugin SDK Channel Route 標準化**：暴露共享頻道路由正規化、解析器驅動目標解析、路由比較工具
- ⭐⭐ **Plugin/Model Catalog Manifest 化**：將 Qianfan、Xiaomi、NVIDIA、Cerebras、Mistral、BytePlus、Fireworks、Together AI 等多家 provider catalog 遷移至 plugin manifest `modelCatalog` rows
- ⭐⭐ **Sandbox/Docker GPU passthrough**：新增選擇性 `sandbox.docker.gpus` 設定，支援 Docker sandbox 容器內 GPU 工作負載（修復 [#57976](https://github.com/openclaw/openclaw/issues/57976)）
- ⭐⭐ **iOS/Android Node Presence**：新增認證 `node.presence.alive` 協議事件及 `node.list` last-seen 欄位（延續 [#63123](https://github.com/openclaw/openclaw/issues/63123)）
- ⭐⭐ **Gateway/Chat 非圖片附件**：`chat.send` 支援非圖片附件，作為 agent 可讀取的媒體路徑 ([#67572](https://github.com/openclaw/openclaw/pull/67572))
- ⭐⭐ **Matrix 串流工具進度**：預設將 tool-progress 更新串流至 Matrix 即時預覽編輯
- ⭐⭐ **Matrix 結構化審批元資料**：附加版本化結構化審批元資料至待審批訊息 ([#72432](https://github.com/openclaw/openclaw/pull/72432))
- ⭐⭐ **Plugin/Models alias 與 suppression**：manifest `modelCatalog.aliases` 及 `modelCatalog.suppressions` 接入模型目錄規劃
- ⭐ **相依性更新**：AWS SDK、Anthropic SDK、AJV、Feishu SDK、tokenjuice 等相依性刷新
- ⭐ **Matrix/QA 審批場景**：新增 exec metadata、chunked fallback、plugin approvals 等即時測試場景
- ⭐ **Apps 相依性刷新**：Peekaboo 3.0.0-beta4、ElevenLabsKit 0.1.1、Commander 0.2.2
- ⭐ **Codex Computer Use 文檔**：文件說明 Codex Computer Use、cua-driver MCP 與 PeekabooBridge 的整合方式

- ⭐⭐⭐ **對外 Proxy 路由**：新增 operator 管理的對外 proxy 路由，含嚴格 HTTP forward-proxy 驗證、loopback-only Gateway 旁路、退出時清理 proxy 狀態 ([#70044](https://github.com/openclaw/openclaw/pull/70044))

- ⭐⭐⭐ **媒體理解 MIME 消毒強化**：收緊媒體理解 MIME 清理，拒絕帶尾隨垃圾的畸形 MIME 值 ([#9795](https://github.com/openclaw/openclaw/issues/9795)、[#72914](https://github.com/openclaw/openclaw/issues/72914))

- ⭐⭐ **Session History 遮蔽持久化**：將遮蔽模式套用至持久化 session transcript 文字，支援跳脫字元類別 ([#42982](https://github.com/openclaw/openclaw/issues/42982))

- ⭐⭐ **Gateway/Auth Loopback Trusted Proxy**：新增 `trustedProxy.allowLoopback` 支援同機 loopback reverse proxy（修復 [#59167](https://github.com/openclaw/openclaw/issues/59167)）

- ⭐ **子系統日誌 Token 遮蔽**：subsystem console sink 遮蔽敏感 token（sk-\* keys、Bearer/Authorization 等）（修復 [#73284](https://github.com/openclaw/openclaw/issues/73284)）
- ⭐ **Discord 群組隱私**：群組/頻道回覆預設保持私密，除非 agent 明確使用 message tool ([#73046](https://github.com/openclaw/openclaw/pull/73046))

### ⭐⭐⭐ Plugin 啟動：Manifest-first 元資料架構
- **用途**: 將 bundled plugin manifests 遷移至顯式 `activation.onStartup` 宣告，Gateway 啟動僅導入有意宣告 startup-time runtime surfaces 的外掛；新增未來模式閘門禁用已棄用的隱式啟動載入；新增相容性警告提示作者遷移；load bundled agent tool-result middleware on demand
- **解決問題**: 過去所有 bundled plugins 在 Gateway 啟動時隱式載入，導致啟動時間過長、不必要的 runtime surface 註冊
- **影響**: Gateway 啟動速度大幅提升，僅載入必要的 plugins，外掛作者可明確宣告啟動行為，為未來移除隱式載入鋪路

### ⭐⭐ 騰訊 Yuanbao 頻道 ([#72756](https://github.com/openclaw/openclaw/pull/72756)、[#73443](https://github.com/openclaw/openclaw/pull/73443))
- **用途**: 註冊 Tencent Yuanbao 外部頻道外掛至官方頻道目錄、contract suites 及社群 plugin 文檔，並新增頻道文檔入口及 `docs/channels/yuanbao.md` 快速上手指南
- **解決問題**: Yuanbao 作為騰訊重要即時通訊平台，過去無法透過 OpenClaw 原生連接

### ⭐⭐ Plugin SDK 通用 Host Hooks ([#72287](https://github.com/openclaw/openclaw/pull/72287))
- **用途**: 新增通用 host hooks 涵蓋 session 狀態、下一回合 context、信任工具策略、UI 描述子、事件排程器清理、及 run-scoped plugin context
- **解決問題**: 外部 plugin 缺乏標準化的 host 互動介面，需要直接存取內部 API
- **影響**: Plugin 開發者可透過正式 SDK hooks 與 host 互動，降低耦合、提升可維護性

### ⭐⭐ Plugin SDK 測試/合約重構
- **用途**: 將大量測試 fixture、contract helper、channel 測試工具從 repo-only `test/helpers` 遷移至 focused SDK subpaths（如 `plugin-sdk/agent-runtime-test-contracts`、`plugin-sdk/channel-test-helpers`、`plugin-sdk/channel-target-testing` 等），並封鎖外部測試匯入 repo-only 橋接
- **解決問題**: 外部 plugin 測試需匯入 repo 內部 `test/helpers` 路徑，版本耦合嚴重
- **影響**: Plugin 測試可使用 focused SDK subpaths，不再依賴 repo 內部結構，提升穩定性及可維護性

### ⭐⭐ Plugin SDK Channel Route 標準化
- **用途**: 暴露共享頻道路由正規化、parser-driven 目標解析、raw-target compact keys、parsed-target types、及路由比較工具至 `openclaw/plugin-sdk/channel-route`，並將 native approval origin matching 遷移至此路由合約
- **解決問題**: 各頻道實作各自的路由解析邏輯，缺乏統一標準
- **影響**: 頻道路由處理統一標準化，降低跨頻道實作差異導致的 bug

### ⭐⭐ Plugin/Model Catalog Manifest 化
- **用途**: 將 Qianfan、Xiaomi、NVIDIA、Cerebras、Mistral、Moonshot、DeepSeek、BytePlus、Volcano Engine、Fireworks、Together AI、Chutes、Kilo、OpenAI、OpenCode Go 等 provider catalogs 遷移至 plugin manifest `modelCatalog` rows
- **解決問題**: Provider catalogs 硬編碼於 runtime 中，難以審計和更新
- **影響**: Provider catalog 可透過 manifest 宣告管理，`models list --all` 無需 runtime normalization

### ⭐⭐ Sandbox/Docker GPU passthrough
- **用途**: 新增選擇性 `sandbox.docker.gpus` passthrough 設定，Docker sandbox 容器可使用 host GPU（當 host Docker runtime 支援 `--gpus`）
- **解決問題**: Docker sandbox 容器無法存取 GPU，限制了需要 GPU 的本地工作負載（如 ML 推論）
- **影響**: 沙箱化 agent 可在需要時使用本地 GPU 資源（修復 [#57976](https://github.com/openclaw/openclaw/issues/57976)）

### ⭐⭐ iOS/Android Node Presence（延續 [#63123](https://github.com/openclaw/openclaw/issues/63123)）
- **用途**: 新增認證 `node.presence.alive` 協議事件及 `node.list` last-seen 欄位，背景 iOS/Android 喚醒可標記配對節點為最近存活
- **解決問題**: 配對節點斷線後失去所有可見性，無法區分「從未連線」與「最近曾連線」
