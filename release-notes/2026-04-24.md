## 重大變更 (Breaking Changes)

### ⭐⭐⭐ Plugin SDK 移除 api.registerEmbeddedExtensionFactory
- **用途**: 移除 Pi-only `api.registerEmbeddedExtensionFactory(...)` 相容路徑，bundled tool-result rewrites 必須改用 `api.registerAgentToolResultMiddleware(...)` 並透過 `contracts.agentToolResultMiddleware` 宣告目標 harnesses
- **解決問題**: Pi 與 Codex app-server dynamic tools 的 tool-result transforms 行為不一致
- **影響**: **使用 `api.registerEmbeddedExtensionFactory(...)` 的 bundled plugins 必須遷移至新 API**，否則 transforms 不會執行

```text
┌──────────────────────────────┐
│  Plugin Tool-result Rewrite  │
└──────────────────────────────┘
               │
               ▼
┌──────────────────────────────┐     ┌──────────────────────────────┐
│  舊 API                      │     │  新 API                      │
│  registerEmbeddedExtension   │     │  registerAgentToolResult     │
│  Factory(...)                │     │  Middleware(...)              │
│  (已移除)                     │     │  + contracts.agentToolResult │
└──────────────────────────────┘     │    Middleware harnesses 宣告  │
                                     └──────────────────────────────┘
                                               │
                                               ▼
                                     ┌──────────────────────────────┐
                                     │  Pi & Codex 統一執行         │
                                     │  transforms                  │
                                     └──────────────────────────────┘
```

### ⭐⭐ Browser 動作逾時設定 ([#62589](https://github.com/openclaw/openclaw/pull/62589))
- **用途**: 新增 `browser.actionTimeoutMs` 設定，預設 60 秒動作預算，避免健康但耗時的 browser 等待在 client transport 邊界失敗
- **解決問題**: 長時間 browser 操作因 transport timeout 而失敗
- **影響**: 複雜頁面操作不再因 timeout 中斷，可依需求調整預算

- **用途**: 支援 `browser.profiles.<name>.headless` 獨立覆蓋，一個 profile 可 headless 而不影響其他 profiles
- **解決問題**: 全域 headless 設定強制所有 browser profiles 同步切換
- **影響**: 可同時維持 headless 自動化 profile 及有頭互動 profile

### ⭐⭐ Gradium TTS Provider ([#64958](https://github.com/openclaw/openclaw/pull/64958))
- **用途**: 新增內建 text-to-speech provider，支援語音筆記及電話輸出
- **解決問題**: 缺乏額外的 TTS provider 選項
- **影響**: 增加語音合成選擇，支援更多使用場景

### ⭐⭐ Control UI Tool Access 重構 ([#71405](https://github.com/openclaw/openclaw/pull/71405))
- **用途**: 精簡即時工具 chips、可折疊工具群組、直接 per-tool toggle、更清晰的 runtime/source provenance
- **解決問題**: Tool Access 面板工具數量多時難以管理
- **影響**: 大量工具的 agent 設定更易管理，直覺的 toggle 操作

### ⭐⭐ Codex Context Engine 整合 ([#70808](https://github.com/openclaw/openclaw/issues/70808), [#70809](https://github.com/openclaw/openclaw/pull/70809))
- **用途**: 在 Codex app-server session 中執行 context-engine bootstrap、assembly、post-turn maintenance 及 engine-owned compaction，保留 native Codex thread state
- **解決問題**: Codex app-server 缺乏 OpenClaw context-engine 的智慧上下文管理
- **影響**: Codex runtime 可使用 OpenClaw 的上下文引擎，同時保持 Codex 原生 thread 可審計

### ⭐⭐ Codex Harness Tool Hooks 橋接 ([#71008](https://github.com/openclaw/openclaw/pull/71008))
- **用途**: 橋接 Codex-native tool hooks 至 OpenClaw plugin hooks 及 approvals，含 bounded relay payloads 及 approval spam protection
- **解決問題**: Codex runtime 的 tool 執行不經過 OpenClaw plugin hooks 及 approval 流程
