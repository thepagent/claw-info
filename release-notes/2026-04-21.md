# OpenClaw v2026.4.21 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21)

## ⚠️ 升級前必讀

### Breaking Changes 摘要

| 項目 | 影響 | 行動 |
|------|------|------|
| owner-enforced commands 改為必須通過 owner identity 驗證 ([#69774](https://github.com/openclaw/openclaw/pull/69774)) | 若你之前依賴 `allowFrom` 萬用規則或空 owner-candidate 清單來讓非 owner 呼叫 owner-only commands，升級後這條路會失效 | 檢查 `enforceOwnerForCommands=true` 的部署，確認真正的 owner identity / owner-candidate 設定正確，不要再依賴寬鬆 fallback |

### 新功能亮點

- **OpenAI image generation 預設改為 `gpt-image-2`**，並同步對齊 live media smoke tests
- **image-generation docs / tool metadata 新增 2K / 4K size hints**，高解析輸出能力更明確
- **多筆穩定性修正**，涵蓋 plugins doctor、Slack thread routing、browser accessibility ref 驗證與 npm install 依賴鏈整理
- **owner-only commands 權限邊界收緊**，避免非 owner 因寬鬆 fallback 誤闖管理指令

## 功能更新 (Features) - by Star Rating

### ⭐⭐⭐ OpenAI image-generation 預設升級為 `gpt-image-2` ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21))
- **用途**: 將 bundled image-generation provider 與 live media smoke tests 的預設模型對齊到 `gpt-image-2`。
- **解決問題**: 舊預設模型與目前 OpenAI 影像生成能力、文件說明和測試基準逐漸脫節，讓使用者與維運者對實際預設行為不夠清楚。
- **影響**: 讓預設影像生成路徑更貼近目前主力模型，也讓 live media smoke tests 與文件敘述一致。

```text
┌────────────────────────────┐
│ Image generation requested │
└────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Bundled provider resolves     │
│ default OpenAI image model    │
└───────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Default now points to         │
│ gpt-image-2                   │
└───────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Runtime docs + smoke tests    │
│ follow the same default path  │
└───────────────────────────────┘
```

### ⭐⭐ OpenAI 2K / 4K size hints 補進 docs 與 tool metadata ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21))
- **用途**: 在 image-generation 文件與工具 metadata 中明確標示更新後的 OpenAI 尺寸提示。
- **解決問題**: 使用者雖可呼叫 image-generation，但高解析尺寸能力若未在文件與 tool surface 明示，常會不知道哪些 size hint 是正式支援路徑。
- **影響**: 對生成高解析圖片的使用者更友善，也讓 agent / tool metadata 對齊實際能力。

## 安全性提升 (Security) - by Star Rating

### ⭐⭐⭐ owner-only commands 改為嚴格驗證 owner identity ([#69774](https://github.com/openclaw/openclaw/pull/69774))
- **用途**: 在 `enforceOwnerForCommands=true` 時，owner-enforced commands 只能由真正的 owner 身分觸發，或由內部 `operator.admin` 路徑觸發。
- **解決問題**: 先前若 `commands.ownerAllowFrom` 未設定，且 `allowFrom` 規則較寬鬆或 owner-candidate 清單為空，非 owner sender 可能仍穿過 fallback 進入 owner-only commands。
- **影響**: 收緊管理指令邊界，避免權限模型因寬鬆 fallback 而被繞過，這是本版最重要的安全變更。

```text
┌────────────────────────────┐
│ Owner-only command arrives │
└────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Check owner enforcement       │
│ and resolve sender identity   │
└───────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Sender is not owner and not   │
│ internal operator.admin       │
└───────────────────────────────┘
              │
              ▼
┌───────────────────────────────┐
│ Reject command instead of     │
│ falling through permissively  │
└───────────────────────────────┘
```

## 錯誤修復 (Bug Fixes) - by Star Rating

### ⭐⭐ Plugins doctor repair 可補 bundled runtime dependencies ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21))
- **用途**: 修正 plugins doctor 從 doctor 路徑補回 bundled plugin runtime dependency 的能力。
- **解決問題**: packaged install 若缺少 channel/provider dependency，先前可能無法透過較精準的 doctor repair 補回，反而逼使用者走較廣泛的 core dependency 安裝。
- **影響**: 套件化安裝的修復路徑更可靠，也更符合最小修補原則。

### ⭐⭐ Image generation fallback 前會先記錄失敗候選 ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21))
- **用途**: 在 provider / model 自動 fallback 前，先把失敗候選寫入 warn log。
- **解決問題**: 先前若後備 provider 成功，前一個失敗候選很可能完全看不到，導致 OpenAI image failure 這類問題難追。
- **影響**: 維運時更容易看見真實失敗路徑，不會因 fallback 成功就失去診斷線索。

### ⭐⭐ Slack thread alias 保留於 runtime outbound sends ([#62947](https://github.com/openclaw/openclaw/pull/62947))
- **用途**: 在 runtime outbound send 時保留呼叫方傳入的 `threadTs`，維持正確 thread alias。
- **解決問題**: generic runtime send 先前可能掉出原本 Slack thread，讓訊息跑到錯誤位置。
- **影響**: Slack thread 內的回覆與自動化送訊更穩定。

### ⭐ Browser 無效 `ax<N>` ref 立即拒絕 ([#69924](https://github.com/openclaw/openclaw/pull/69924))
- **用途**: 對無效 accessibility ref 在 act path 立即報錯。
- **解決問題**: 先前這類錯誤常要等到 browser action timeout 才爆出來，拖慢除錯。
- **影響**: browser automation 錯誤更快暴露，節省等待時間。

### ⭐ npm install 的 deprecated dependency chain 噪音下降 ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.21))
- **用途**: 將 `node-domexception` alias 鏡射到 root `package.json` overrides。
- **解決問題**: npm install 會沿著 `google-auth-library -> gaxios -> node-fetch -> fetch-blob -> node-domexception` 鏈顯示 deprecated 噪音。
- **影響**: 安裝輸出更乾淨，也減少對依賴健康度的誤判。
