- **GPT-5.4-pro 前向相容** — 新增 gpt-5.4-pro 模型支援，含 Codex 定價/限制與清單可見性（[#66453](https://github.com/openclaw/openclaw/pull/66453)）

## 功能更新 (Features) - by Star Rating

### ⭐⭐ OpenAI Codex gpt-5.4-pro 前向相容支援 ([#66453](https://github.com/openclaw/openclaw/pull/66453))
- **用途**: 在 Codex provider 中新增 gpt-5.4-pro 模型定義，包含定價、token 限制與 list/status 可見性。
- **解決問題**: gpt-5.4-pro 發佈後，OpenClaw 使用者無法直接選用該模型，需手動設定 provider catalog。

- **影響**: agent 在 Telegram 論壇中能根據主題名稱調整回覆行為，提升上下文感知能力。

## 安全性提升 (Security) - by Star Rating

### ⭐⭐⭐ Control UI ReDoS 防護：以 markdown-it 取代 marked.js ([#46707](https://github.com/openclaw/openclaw/pull/46707))
- **用途**: 將 Control UI 的 Markdown 渲染引擎從 marked.js 換成 markdown-it，消除 ReDoS 攻擊面。
- **解決問題**: 惡意構造的 Markdown 內容可透過 marked.js 的正規表達式回溯凍結整個 Control UI。
- **影響**: Control UI 不再受 Markdown ReDoS 攻擊影響，對外開放的實例安全性顯著提升。

```text
┌─────────────────────────┐
│ 使用者送出 Markdown 內容 │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────────┐
│ 舊路徑: marked.js 解析      │
│ → 惡意 pattern 觸發回溯     │
│ → UI 凍結 (ReDoS)          │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 新路徑: markdown-it 解析    │
│ → 無災難性回溯風險          │
│ → UI 正常渲染              │
└─────────────────────────────┘
```

### ⭐⭐⭐ Browser SSRF 政策擴展至 snapshot / screenshot / tab 路由 ([#66040](https://github.com/openclaw/openclaw/pull/66040))
- **用途**: 將現有的 browser SSRF 防護政策一致套用到 snapshot、screenshot 與 tab 管理路由。
- **解決問題**: 這些路由先前未受 SSRF 政策保護，可能被利用來探測內部網路或存取受限資源。
- **影響**: browser 功能的所有網路存取路徑現在都受到統一的 SSRF 防護。

```text
│ ├─ 受限 hostname？        │
│ └─ 違反 navigation mode？ │
└──────────┬───────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌─────────┐ ┌─────────┐
│  允許   │ │  拒絕   │
│  執行   │ │  請求   │
└─────────┘ └─────────┘
```

### ⭐⭐⭐ Gateway tool 拒絕啟用 security audit flags 的設定變更 ([#62006](https://github.com/openclaw/openclaw/pull/62006))
- **用途**: 攔截 agent 透過 gateway tool 的 config.patch / config.apply 呼叫中會新啟用 security audit flags 的變更。
- **解決問題**: 模型驅動的設定變更若能開啟 security audit flags，等於讓 agent 自行提升安全稽核權限。
- **影響**: 防止 agent 透過設定變更繞過安全邊界，operator 設定的安全旗標不會被 agent 側改寫。

```text

           │

```

### ⭐⭐⭐ 本地附件路徑 fail closed ([#66022](https://github.com/openclaw/openclaw/pull/66022))
- **用途**: 當本地附件路徑無法透過 realpath 正規化解析時，直接拒絕處理而非嘗試繼續。
- **解決問題**: 若路徑含 symlink 或不存在的中間目錄，繼續處理可能導致路徑穿越或存取非預期檔案。
- **影響**: 附件處理路徑更安全，消除路徑穿越風險。

```text
┌──────────────────────────┐
│ 收到本地附件路徑          │
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 嘗試 realpath() 正規化   │
└──────────┬───────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌──────────┐ ┌──────────────┐
│ 成功解析 │ │ 解析失敗     │
│ → 繼續   │ │ → fail closed│
│ 處理     │ │   拒絕附件   │
└──────────┘ └──────────────┘
```

### ⭐⭐ Heartbeat 對不受信任的 hook:wake 事件強制 owner downgrade ([#66031](https://github.com/openclaw/openclaw/pull/66031))
- **用途**: 當 heartbeat 收到 untrusted hook:wake 系統事件時，強制將執行身份降級為非 owner。
- **解決問題**: 不受信任的 wake 事件若以 owner 身份執行，可能觸發高權限操作。
- **影響**: heartbeat 事件處理的權限邊界更嚴謹。

- **用途**: 依 user/chat kind 正規化 Feishu allowlist 條目。

### ⭐ Doctor/systemd 防止 secrets 重新嵌入 ([#66249](https://github.com/openclaw/openclaw/pull/66249))

- **影響**: systemd 部署的 secrets 管理更安全。

## 錯誤修復 (Bug Fixes) - by Star Rating

### ⭐⭐⭐ Gateway sessions 防止 heartbeat/cron/exec turns 覆寫 shared-session routing ([#66073](https://github.com/openclaw/openclaw/pull/66073), [#63733](https://github.com/openclaw/openclaw/pull/63733), [#35300](https://github.com/openclaw/openclaw/pull/35300))
- **用途**: 阻止 heartbeat、cron-event、exec-event 類型的 turn 覆寫 shared-session 的 routing 資訊。
- **解決問題**: 系統事件 turn 若覆寫 routing，會導致後續使用者訊息被路由到錯誤的 channel 或 session。
- **影響**: 多通道共用 session 的路由穩定性大幅提升，解決長期存在的間歇性路由錯亂問題。

```text
┌──────────────────────────┐
│ Shared session 收到      │
│ heartbeat/cron/exec turn │
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────────┐
│ 檢查 turn 類型               │
│ 是否為系統事件？              │
└──────────┬───────────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌──────────┐ ┌────────────────┐
│ 使用者   │ │ 系統事件       │
│ 訊息     │ │ → 跳過 routing │
│ → 正常   │ │   覆寫         │
│ 更新     │ └────────────────┘
└──────────┘
```

### ⭐⭐⭐ Cron scheduler 不再於無有效未來時段時發明短重試 ([#66019](https://github.com/openclaw/openclaw/pull/66019), [#66083](https://github.com/openclaw/openclaw/pull/66083))
- **用途**: 當 cron next-run 計算回傳無有效未來時段時，停止產生不合理的短間隔重試。
- **解決問題**: 無效的短重試會造成 cron scheduler 高頻空轉，浪費資源並產生大量無意義日誌。
- **影響**: cron 排程器在邊界情況下的行為更合理，不再產生資源浪費。

```text
┌──────────────────────────┐
│ Cron job 觸發 next-run   │
│ 計算                     │
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────────┐
│ 計算結果：無有效未來時段      │
└──────────┬───────────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌──────────┐ ┌────────────────┐
│ 舊行為   │ │ 新行為         │
│ → 發明   │ │ → 停止排程     │
│ 短重試   │ │   等待下次     │
│ → 空轉   │ │   有效觸發     │
└──────────┘ └────────────────┘
```

### ⭐⭐ Context engine 拒絕 info.id 不符的 plugin engines ([#63222](https://github.com/openclaw/openclaw/pull/63222))
- **用途**: 當 resolved plugin engine 的 info.id 與 registered slot id 不符時，拒絕載入該 engine。
- **解決問題**: 不一致的 engine ID 可能導致 context engine 行為不可預測或衝突。
- **影響**: plugin engine 的註冊驗證更嚴格，降低 runtime 異常風險。

### ⭐⭐ Memory/Ollama 恢復內建 embedding adapter ([#63429](https://github.com/openclaw/openclaw/pull/63429), [#66078](https://github.com/openclaw/openclaw/pull/66078), [#66163](https://github.com/openclaw/openclaw/pull/66163))
- **用途**: 恢復 memory-core 中內建的 ollama embedding adapter。
- **解決問題**: adapter 遺失導致使用 Ollama 做 memory embedding 的使用者完全無法使用記憶功能。
- **影響**: Ollama 記憶功能恢復正常運作。

### ⭐⭐ Auto-reply sendPolicy: deny 不再阻擋 inbound 處理 ([#65461](https://github.com/openclaw/openclaw/pull/65461), [#53328](https://github.com/openclaw/openclaw/pull/53328))
- **用途**: 修正 sendPolicy: deny 設定不再錯誤地阻擋 inbound 訊息的處理流程。
- **解決問題**: deny 政策本意是阻止 outbound 發送，但錯誤地連 inbound 處理也一起擋了。
- **影響**: 使用 sendPolicy: deny 的使用者可以正常接收和處理訊息。

### ⭐⭐ Agents/OpenAI reasoning-only 或空 turn 的 bounded continuation ([#66167](https://github.com/openclaw/openclaw/pull/66167))
- **用途**: 當 GPT-style embedded run 產生 reasoning-only 或空 turn 時，正確恢復並進行 bounded continuation。
- **解決問題**: 這類 turn 先前可能導致 agent 卡住或無限迴圈。
- **影響**: GPT-5.4 系列模型的 turn 處理更穩定。

### ⭐⭐ Browser/CDP managed Chrome readiness 繞過 SSRF policy ([#65695](https://github.com/openclaw/openclaw/pull/65695), [#66043](https://github.com/openclaw/openclaw/pull/66043))
- **用途**: 讓 managed local Chrome 的 readiness 檢查繞過 browser SSRF policy。
- **解決問題**: strict SSRF 預設會阻擋 loopback 連線，導致本地 Chrome 啟動檢查失敗。
- **影響**: 使用 managed Chrome 的 browser 功能在 strict SSRF 模式下恢復正常。

### ⭐⭐ Cron scheduler 維護修復保留 error-backoff floor ([#66019](https://github.com/openclaw/openclaw/pull/66019), [#66083](https://github.com/openclaw/openclaw/pull/66083), [#66113](https://github.com/openclaw/openclaw/pull/66113))
- **用途**: 當維護修復重算遺失的 next-run 時，保留現有的 error-backoff floor。
- **解決問題**: 重算若清除 backoff floor，會讓持續失敗的 job 立即重試而非等待退避。
- **影響**: cron 錯誤恢復行為更合理，避免失敗 job 的重試風暴。

### ⭐⭐ Agents/OpenAI minimal thinking 對應 low reasoning effort ([GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.14))
- **用途**: 將 minimal thinking 設定正確對應到 OpenAI 支援的 low reasoning effort for GPT-5.4。
- **解決問題**: 未正確對應會導致 GPT-5.4 忽略 thinking 設定或使用錯誤的 effort level。
- **影響**: GPT-5.4 的 thinking 行為更符合使用者預期。

### ⭐⭐ Agents/context engine 從首次 tool-loop delta 開始壓縮 ([#63555](https://github.com/openclaw/openclaw/pull/63555))
- **用途**: 讓 engine-owned sessions 從第一次 tool-loop delta 就開始進行 compaction。
- **解決問題**: 延遲壓縮會讓長 tool-loop 的 session 快速膨脹。
- **影響**: 長對話與多工具呼叫場景的記憶體使用更可控。

### ⭐⭐ Discord native /status 回傳真實狀態卡 ([#54629](https://github.com/openclaw/openclaw/pull/54629))
- **用途**: 讓 Discord native /status 互動回傳真實的狀態卡片而非空回應。
- **解決問題**: /status 指令先前可能回傳空白或錯誤格式的回應。
