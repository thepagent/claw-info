## 安全性提升 (Security) - by Star Rating

### ⭐⭐ Plugin Install Scanner False Positive 修復

- **用途**：收緊 `env-harvesting` 規則的上下文比對邏輯，要求 `process.env` 存取與網路傳送（`fetch`/`http.request` 等）必須在 **同一 8 行視窗內** 才觸發警示，而非整個檔案任意距離皆觸發
- **解決問題**：官方打包的 bundled plugin（如 Discord provider）在編譯後的單一檔案中同時包含無害的 `process.env` 使用（如 `resolvePreferencesStorePath`）與網路傳送（如 `sendMessage`），兩者在原始碼中相隔數十行以上、毫無關聯，但舊掃描器因缺乏距離限制而將其標記為潛在憑證竊取，導致安裝被封鎖
- **影響**：官方 bundled plugins 在安裝時不再被錯誤封鎖，同時仍能偵測真正的惡意套件（`process.env` 與網路傳送在同一小範圍內）

```text
┌──────────────────────────────────┐
│  Plugin 安裝請求                  │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  skill-scanner 掃描              │
│  scanSource(source, filePath)    │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  env-harvesting 規則比對         │
│  process.env + 網路傳送          │
└──────────────┬───────────────────┘
          ┌────┴────────┐
          │             │
          ▼             ▼
   距離 ≤ 8 行      距離 > 8 行
   （同上下文）     （無關聯程式碼）
          │             │
          ▼             ▼
   ⚠️ 可能惡意     ✅ 安全通過
   （封鎖安裝）   （允許安裝）
```

## 錯誤修復 (Bug Fixes) - by Star Rating

### ⭐⭐ Plugin Install Scanner Bundled Plugin 誤判修復

- **用途**：在 `SourceRule` 新增 `requiresContextWindowLines: 8` 屬性，提取 `findSourceRuleMatch()` 函式，以逐行滑動視窗檢查主要比對（`process.env`）與次要上下文（網路傳送 pattern）是否在 8 行範圍內
- **解決問題**：v2026.5.3 的 Plugin Install Scanner 使用全檔案範圍的上下文比對，導致官方 bundled plugins（經 webpack/esbuild 等工具編譯為單一檔案）中距離遙遠的無害 `process.env` 存取與網路傳送被錯誤關聯，安裝被拒絕
- **影響**：`openclaw@2026.5.3-1` 的 bundled plugins 可正常安裝，惡意套件偵測能力不受影響
