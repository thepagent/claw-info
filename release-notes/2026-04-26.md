### 功能更新 (Features)

- ⭐ **WebChat/Control UI：非影片檔案上傳**：支援非影片檔案附件上傳 ([#70947](https://github.com/openclaw/openclaw/pull/70947))

### ⭐⭐⭐ Control UI/Talk：Google Live 瀏覽器語音對話
- **用途**: 新增通用瀏覽器即時傳輸協定、Google Live browser Talk sessions 含受控短期 token，以及 Gateway relay 用於後端即時語音 plugin
- **解決問題**: 過去瀏覽器 Talk session 缺乏統一的即時傳輸協定，且無法透過 Gateway relay 執行後端語音 plugin
- **影響**: 使用者可在瀏覽器中直接使用 Google Live 語音對話，Gateway relay 讓後端即時語音 plugin 得以安全運作，同時修復了 WebRTC fallback、stale voice button 等多項問題

### ⭐⭐⭐ Agents/compaction：大型 transcript 預先壓縮
- **用途**: 新增 `agents.defaults.compaction.maxActiveTranscriptBytes` opt-in preflight trigger，當活動 JSONL 過大時自動執行壓縮，需要 transcript rotation 讓成功壓縮將後續 turns 移至較小的 successor 檔案
- **解決問題**: 長時間運行的 session 其 JSONL 檔案不斷增長，超過 provider context 限制後導致錯誤或效能下降
- **影響**: 長 session 自動壓縮歷史，使用更小的 successor 檔案繼續運行，避免 raw byte-splitting 歷史記錄

```text

          ▼

└──────────────────────┘               │
                                        ▼
                              ┌──────────────────────┐
```

- **解決問題**: 過去缺乏直觀的設定變更差異檢視，難以追蹤 pending config 變更

- **用途**: 最佳化快速設定 dashboard 格線，讓常用卡片在桌面、平板、手機版面一致對齊，不浪費水平空間
- **解決問題**: Dashboard 卡片在不同裝置上排列不一致，水平空間利用不佳
- **影響**: 統一的跨裝置 dashboard 體驗

### ⭐⭐ Ollama/memory 查詢前綴 ([#45013](https://github.com/openclaw/openclaw/pull/45013))
- **用途**: 為 `nomic-embed-text`、`qwen3-embedding`、`mxbai-embed-large` 新增模型專屬的 memory-search 查詢前綴，文件批次保持不變
- **解決問題**: 不同嵌入模型對查詢有最佳前綴需求，缺少前綴導致檢索品質下降
- **影響**: Ollama 記憶搜尋品質提升，同時保持文件嵌入不變

### ⭐⭐ Plugin install：分層 runtime-deps ([#72396](https://github.com/openclaw/openclaw/pull/72396))
- **用途**: 允許 `OPENCLAW_PLUGIN_STAGE_DIR` 包含分層 runtime-dependency roots，在安裝前解析唯讀預裝 deps，僅將缺失 deps 安裝至最終可寫 root
- **解決問題**: 唯讀預裝 deps 與新安裝 deps 衝突，導致安裝失敗
- **影響**: Plugin 安裝流程更穩健，支援分層依賴架構

### ⭐⭐ Plugin manifest 遷移
- **用途**: 將 pre-runtime model-id normalization、provider endpoint host metadata 及 OpenAI-compatible request-family hints 移至 plugin manifest，core 不再攜帶 bundled-provider routing tables
- **解決問題**: Core 包含大量 bundled-provider 路由表，增加維護負擔和耦合
- **影響**: Provider 路由邏輯移至 manifest 層級，core 更精簡，plugin 自治性更高

### ⭐ CLI/models：provider-filtered model listing
- **用途**: 透過明確 source plan 路由 provider-filtered 模型列表，user config、installed manifest rows、Provider Index previews 及 scoped runtime fallbacks 保持穩定權威順序
- **解決問題**: 模型列表權威順序不一致，可能導致非預期的模型選擇
- **影響**: 模型列表來源優先順序穩定可預測

### ⭐ Nodes/CLI：`openclaw nodes remove`
- **用途**: 新增 `openclaw nodes remove --node <id|name|ip>` 及 `node.pair.remove` RPC
- **解決問題**: 過時的 node 配對記錄需手動編輯 state 檔案才能清理
- **影響**: 可透過 CLI/RPC 輕鬆清理過時 node 配對

### ⭐ Memory-core/dreaming：`dreaming.model` 設定 ([#65963](https://github.com/openclaw/openclaw/pull/65963))
- **用途**: 新增 `dreaming.model` 設定旋鈕，讓 Dream Diary 叙事 subagent 可指定使用的模型
- **解決問題**: Dream Diary subagent 無法獨立設定模型
- **影響**: 叙事生成可使用專用模型，提升彈性

### ⭐ Onboarding/GitHub Copilot
- **用途**: 新增 `--github-copilot-token` 支援非互動設定，含 env fallback、tokenRef storage、saved-profile reuse 及 default-model wiring ([#50002](https://github.com/openclaw/openclaw/pull/50002))
- **解決問題**: GitHub Copilot 設定缺乏非互動/自動化路徑
- **影響**: 自動化部署可使用 token flag 完成 Copilot 設定

### ⭐ Gateway/install：`--wrapper` 服務安裝路徑 ([#69400](https://github.com/openclaw/openclaw/pull/69400), [#72445](https://github.com/openclaw/openclaw/pull/72445))
