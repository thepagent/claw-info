# OpenClaw v2026.4.25 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.4.25)

## 概述

### 功能更新 (Features)
- ⭐⭐⭐ **TTS 全面升級**：`/tts latest` 朗讀支援、chat-scoped auto-TTS、personas、per-agent/per-account overrides，新增 Azure Speech、Xiaomi MiMo、Local CLI、Inworld、Volcengine、ElevenLabs v3 等提供者 ([#51776](https://github.com/openclaw/openclaw/pull/51776), [#55614](https://github.com/openclaw/openclaw/pull/55614), [#56239](https://github.com/openclaw/openclaw/pull/56239), [#55972](https://github.com/openclaw/openclaw/pull/55972), [#55641](https://github.com/openclaw/openclaw/pull/55641), [#68321](https://github.com/openclaw/openclaw/pull/68321), [#70748](https://github.com/openclaw/openclaw/pull/70748))
- ⭐⭐⭐ **Plugin 啟動遷移至 Cold Persisted Registry**：plugin 啟動、安裝、探索全面遷移至冷持久化註冊表，消除啟動時的廣泛 manifest 掃描
- ⭐⭐⭐ **OpenTelemetry 大幅擴展**：涵蓋 model calls、token usage、tool loops、harness runs、exec processes、outbound delivery、context assembly、memory pressure
- ⭐⭐⭐ **Voice Wake Trigger Routing**：macOS 語音喚醒詞可路由至指定 agent/session 目標 ([#30354](https://github.com/openclaw/openclaw/pull/30354))
- ⭐⭐ **Control UI PWA/Web Push**：Gateway chat 支援 PWA 安裝及 Web Push 通知 ([#44590](https://github.com/openclaw/openclaw/pull/44590))
- ⭐⭐ **Browser 自動化強化**：safe tab URLs、iframe-aware role snapshots、CDP readiness tuning、headless one-shot launch、browser doctor deep probing
- ⭐⭐ **CLI/Crestodian TUI Setup**：首 Run setup helper、本地 planner fallback、完整 TUI 互動、context mode selector ([#71720](https://github.com/openclaw/openclaw/pull/71720), [#71760](https://github.com/openclaw/openclaw/pull/71760))
- ⭐⭐ **LiteLLM Image Generation Provider**：`image_generate model=litellm/...` 透過 LiteLLM proxy 路由 ([#70246](https://github.com/openclaw/openclaw/pull/70246))
- ⭐⭐ **MiniMax Portal Music/Video**：`minimax-portal` 支援音樂與影片生成，保留 OAuth 與區域端點 ([#63241](https://github.com/openclaw/openclaw/pull/63241))
- ⭐⭐ **Google Meet Attendance Export**：行事曆驅動的出席匯出、dry-run 預覽、工具平價

- ⭐⭐ **Discord Voice Model Override**：語音頻道可使用獨立 LLM 模型 ([#64368](https://github.com/openclaw/openclaw/pull/64368))

- ⭐ **CLI Image Generation --background**：通用 `--background` 參數及 fal `--output-format` 支援
- ⭐ **Plugin Hooks 擴展**：before-agent-finalize hooks、cron jobId context、Codex MCP hook relay ([#71765](https://github.com/openclaw/openclaw/pull/71765), [#71758](https://github.com/openclaw/openclaw/pull/71758), [#71707](https://github.com/openclaw/openclaw/pull/71707))
- ⭐ **SenseAudio Batch Transcription**：新增 SenseAudio 批次音訊轉錄提供者 ([#66943](https://github.com/openclaw/openclaw/pull/66943))
- ⭐ **Provider Index Preview**：未安裝 provider 可從 Provider Index 取得安裝 metadata
- ⭐ **ACPX Droid Live Bind**：Factory Droid 加入 ACP Docker bind 矩陣

### 安全性提升 (Security)
- ⭐⭐⭐ **Session Transcript Redaction**：設定 redaction patterns 套用至持久化 session transcript，防止敏感文字明文儲存 (Fixes [#42982](https://github.com/openclaw/openclaw/issues/42982))
- ⭐⭐⭐ **Gateway Device Token Scope Containment**：強制 token rotation/revocation 的呼叫者範圍隔離，pairing-only session 無法變更高權限 operator token (Fixes [#71990](https://github.com/openclaw/openclaw/issues/71990))
- ⭐⭐⭐ **Release Package Integrity**：拒絕預填充的 bundled plugin `.openclaw-install-stage` 目錄，防止 release tarball 夾帶惡意 runtime-dep 殘留 (Fixes [#71752](https://github.com/openclaw/openclaw/issues/71752))
- ⭐⭐ **Logging Redaction at Sink Exit**：console/file-log 出口套用 secret pattern redaction (Fixes [#67953](https://github.com/openclaw/openclaw/issues/67953))
- ⭐⭐ **Gateway Service Version Guard**：拒絕舊版 OpenClaw binary 變更新版 gateway 服務 (Fixes [#57079](https://github.com/openclaw/openclaw/issues/57079))
- ⭐⭐ **OTEL Bounded Diagnostics**：所有 OTEL spans/metrics 使用低基數屬性，不含 session/prompt/response 內容
- ⭐⭐ **Media MIME/ZIP Preflight Bounding**：限制未受信任附件和 ClawHub archives 的 MIME sniffing 和 ZIP preflight 資源消耗
- ⭐⭐ **Inbound Message Dedupe Poison Recovery**：重播不安全的 provider/runtime 失敗後投毒去重，防止重試產生重複訊息 (Fixes [#69303](https://github.com/openclaw/openclaw/issues/69303))
- ⭐ **Security Audit on Cold Metadata**：channel exposure 和 plugin allowlist 所有權檢查基於唯讀 plugin index，不需載入 runtime
- ⭐ **Subagent Loopback Auth**：internal RPC 使用 shared gateway token 認證，避免觸發不必要的 pairing prompt (Fixes [#63548](https://github.com/openclaw/openclaw/issues/63548))
- ⭐ **Memory-host SDK SSRF DNS Pinning**：trusted env-proxy 模式保留 SSRF DNS pinning (Fixes [#52162](https://github.com/openclaw/openclaw/issues/52162))
- ⭐ **Slack Edit Spoofing Prevention**：防止已編輯 bot message 中的 user mention 偽造 DM 發送者 ([#71700](https://github.com/openclaw/openclaw/pull/71700))
- ⭐ **Unsafe Remote MEDIA URL Rejection**：拒絕不安全的遠端 `MEDIA:` URL

### ⭐⭐⭐ Plugin 啟動遷移至 Cold Persisted Registry
- **用途**: Gateway 啟動 plugin 規劃遷移至版本化冷持久化註冊表索引；provider discovery、provider ownership、catalog hook、setup metadata 全面從 manifest 掃描遷移至 cold registry；`plugins list` 預設讀取 cold snapshot；`plugins registry --refresh` 提供顯式修復
- **解決問題**: 每次啟動需要廣泛掃描 plugin manifest 目錄，大型 registry 在慢速主機上造成顯著延遲；config、status、doctor、capabilities 等唯讀路徑也會觸發 runtime 載入
- **影響**: 啟動速度大幅提升，唯讀路徑（config、status、doctor、audit）不再載入 plugin runtime；`OPENCLAW_DISABLE_PERSISTED_PLUGIN_REGISTRY` 標記為 deprecated break-glass switch

```text
┌──────────────────────┐     ┌──────────────────────┐
│  Legacy: 啟動時掃描  │     │  New: Cold Registry  │
│  所有 plugin manifest│     │  索引快照             │
└──────────────────────┘     └──────────────────────┘
         │                            │
         ▼                            ▼
┌──────────────────────┐     ┌──────────────────────┐
│  每次讀取載入        │     │  唯讀路徑直接查索引  │
│  plugin runtime      │     │  (不載入 runtime)    │
└──────────────────────┘     └──────────────────────┘
                                      │
                                      ▼
                              ┌──────────────────────┐
                              │  需要執行時才載入    │
                              │  runtime module      │
                              └──────────────────────┘
```

### ⭐⭐⭐ OpenTelemetry 大幅擴展 ([#71471](https://github.com/openclaw/openclaw/pull/71471), [#71451](https://github.com/openclaw/openclaw/pull/71451), [#71450](https://github.com/openclaw/openclaw/pull/71450))
- **用途**: OTEL 覆蓋範圍擴展至 model calls（`gen_ai.client.token.usage` histogram、`gen_ai.client.operation.duration` histogram）、tool loops（`openclaw.tool.loop` counters/spans）、harness runs（`openclaw.harness.run` spans）、exec processes（`openclaw.exec` spans）、outbound delivery lifecycle、context assembly diagnostics、memory pressure histograms；所有 spans/metrics 使用 bounded low-cardinality attributes，不含 session/prompt/response 內容；支援 signal-specific OTLP endpoint overrides；W3C `traceparent` header propagation

### ⭐⭐⭐ Voice Wake Trigger Routing ([#30354](https://github.com/openclaw/openclaw/pull/30354))
- **用途**: macOS 語音喚醒詞可選擇路由至配置的 agent 或 session 目標，含 Gateway routing API 和 node update events
- **解決問題**: 過去 voice wake 只能觸發預設 agent，無法根據喚醒詞選擇不同 agent/session
- **影響**: 使用者可配置多個喚醒詞分別路由至不同 agent，實現語音喚醒的個人化分流

└──────────────────────┘     └──────────────────────┘
                                      │
                                      ▼

### ⭐⭐ Control UI PWA/Web Push ([#44590](https://github.com/openclaw/openclaw/pull/44590))
- **用途**: Control UI 支援 PWA 安裝至桌面/行動裝置，並透過 Web Push API 推送 Gateway chat 通知
- **解決問題**: 過去 Control UI 僅能在瀏覽器分頁中使用，關閉分頁後無法接收通知
- **影響**: 使用者可將 Control UI 安裝為獨立應用，接收推播通知，提升即時互動體驗

### ⭐⭐ Browser 自動化強化
- **用途**: agent 回覆中提供 safe tab URLs；新增 CDP-native role snapshot fallback 含 iframe-aware refs、cursor-clickable detection、target attach preparation；`openclaw browser doctor --deep` 提供 live snapshot probing；`openclaw browser start --headless` 提供 one-shot 無頭啟動；本地 managed Chrome launch discovery 和 CDP readiness timeout 可調整（支援 Raspberry Pi 等慢速主機）
- **解決問題**: 瀏覽器自動化在複雜頁面（含 iframe）中 snapshot 不準確；CDP readiness 在慢速主機上超時；無法快速啟動無頭瀏覽器
- **影響**: iframe 頁面操作更可靠；慢速主機可調整 timeout；`--headless` 提供便捷的無頭模式切換；`--deep` 診斷提供更完整的瀏覽器健康報告

### ⭐⭐ CLI/Crestodian TUI Setup ([#71720](https://github.com/openclaw/openclaw/pull/71720), [#71760](https://github.com/openclaw/openclaw/pull/71760))
- **用途**: 新增首 Run setup helper、本地 planner fallback、完整 TUI 互動式 Crestodian、啟動進度指示器、context mode selector、更短的啟動問候語
- **解決問題**: 首次設定流程缺乏引導，TUI 體驗不完整，context mode 選擇不明確
- **影響**: 新使用者可透過互動式 TUI 完成初始設定；context mode 選擇更清晰；啟動體驗更流暢

### ⭐⭐ LiteLLM Image Generation Provider ([#70246](https://github.com/openclaw/openclaw/pull/70246))
- **用途**: 註冊 `litellm` 作為 image-generation provider，使 `image_generate model=litellm/...` 呼叫和 fallbacks 可透過 LiteLLM proxy 路由
- **解決問題**: 使用 LiteLLM proxy 的使用者無法透過 OpenClaw 統一介面使用 image generation
- **影響**: `image_generate` 和 `agents.defaults.imageGenerationModel.fallbacks` 可使用 LiteLLM 路由

### ⭐⭐ MiniMax Portal Music/Video ([#63241](https://github.com/openclaw/openclaw/pull/63241))
- **用途**: 註冊 `minimax-portal` 用於 music 和 video generation，保留 OAuth 認證和區域 MiniMax base URL
- **解決問題**: MiniMax 的音樂/影片生成需要獨立設定，缺少統一入口
- **影響**: 透過 `music_generate` 和 `video_generate` 工具可直接使用 MiniMax 服務

### ⭐⭐ Google Meet Attendance Export
- **用途**: 新增行事曆驅動的出席匯出工作流程、export manifests、dry-run 預覽、meeting records 工具平價
- **解決問題**: 無法從 Google Meet 匯出出席記錄進行後續分析
- **影響**: 可自動化匯出會議出席資料，含 dry-run 預覽確保準確性

### ⭐⭐ Codex Harness MCP Hooks ([#71765](https://github.com/openclaw/openclaw/pull/71765), [#71758](https://github.com/openclaw/openclaw/pull/71758))
- **用途**: Codex harness 要求 app-server `0.125.0+`，並透過 OpenClaw hook relay 支援 native MCP `PreToolUse`、`PostToolUse`、`PermissionRequest` payloads
- **解決問題**: Codex 原生 MCP hooks 無法透過 OpenClaw plugin 系統傳遞
- **影響**: Codex 的工具使用前後和權限請求可被 OpenClaw plugin hook 攔截和處理
