## 功能更新 (Features) - by Star Rating

### ⭐⭐⭐ File-Transfer Plugin ([#74742](https://github.com/openclaw/openclaw/pull/74742))

- **用途**：新增內建 `file-transfer` 插件，提供 `file_fetch`、`dir_list`、`dir_fetch`、`file_write` 四項 agent 工具，讓 agent 可對已配對節點執行二元檔案操作
- **解決問題**：agent 缺乏對遠端節點的檔案系統存取能力，無法直接讀寫或列出目錄
- **影響**：安全地擴展 agent 至檔案操作領域，內建 per-node 預設拒絕路徑策略與 16 MB 傳輸上限

```text
┌──────────────────┐     ┌──────────────────┐
│  Agent Request   │────►│  File-Transfer   │
│  (tool call)     │     │  Plugin          │
└──────────────────┘     └────────┬─────────┘
                                  │
                         ┌────────┴────────┐
                         │  Per-Node Path  │
                         │  Policy Check   │
                         │  (default-deny) │
                         └────────┬────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
              ┌──────────┐ ┌──────────┐ ┌──────────┐
              │file_fetch│ │dir_list/ │ │file_write│
              │(read)    │ │dir_fetch │ │(write)   │
              └──────────┘ │(list)    │ └──────────┘
                           └──────────┘
                                 │
                                 ▼
                        ┌──────────────┐
                        │  Symlink     │
                        │  Traversal   │
                        │  Refused     │
                        │  (opt-in)    │
                        └──────────────┘
```

### ⭐⭐⭐ Streaming Progress 統一框架 ([#76327](https://github.com/openclaw/openclaw/pull/76327))

- **用途**：新增 `streaming.mode: "progress"` 統一 draft 框架，跨 Discord、Telegram、Matrix、Slack、Microsoft Teams 共用 auto 單字狀態標籤與 progress 設定
- **解決問題**：各頻道 progress streaming 實作不一致，缺乏統一配置介面
- **影響**：operator 可在 config 中統一設定 progress draft 行為，所有頻道行為一致

```text
┌──────────────────────┐
│  streaming.mode:     │
│  "progress"          │
└──────────┬───────────┘
           │
     ┌─────┴──────────────────────┐
     │                            │
     ▼                            ▼
┌─────────────┐           ┌──────────────┐
│ Auto Label  │           │ Config       │
│ (單字狀態)   │           │ • maxLines   │
│             │           │ • label      │
│             │           │ • toolProgress│
└──────┬──────┘           └──────┬───────┘
       │                         │
       └─────────┬───────────────┘
                 │
    ┌────────────┼──────────────────────────┐
    │            │            │              │
    ▼            ▼            ▼              ▼
┌────────┐ ┌─────────┐ ┌────────┐  ┌────────────┐
│Discord │ │Telegram │ │Matrix  │  │Slack/Teams │
│Draft   │ │Preview  │ │Progress│  │Progress    │
└────────┘ └─────────┘ └────────┘  └────────────┘
```

### ⭐⭐⭐ `/steer` 指令 ([#76934](https://github.com/openclaw/openclaw/pull/76934))

- **用途**：新增 `/steer <message>` 指令，可對運行中的 session 發送方向調整，不建立新回合也不受佇列獨立性影響
- **解決問題**：調整運行中 agent 方向需發送新訊息（建立新回合），或在 idle 時無法引導
- **影響**：operator 可即時微調運行中的 agent 行為，不干擾現有佇列狀態

```text
┌──────────────────┐
│  /steer <msg>    │
└────────┬─────────┘
         │
         ▼
┌──────────────────────┐
│  Session 狀態檢查     │
└──────────┬───────────┘
      ┌────┴────┐
      │         │
      ▼         ▼
   運行中     Idle
      │         │
      ▼         ▼
 注入 steering  佇列化等待
 到當前回合      下次運行時套用
 (不建立新回合)
```

### ⭐⭐ `/side` 別名

- **用途**：新增 `/side` 作為 `/btw` 側邊提問的文字與原生斜線指令別名
- **解決問題**：`/btw` 名稱不夠直覺
- **影響**：更自然的指令名稱，行為與 `/btw` 完全相同

### ⭐⭐ Google Meet Realtime Voice 完整修復

- **用途**：修復 Google Meet 即時語音的 Chrome 媒體權限授予、local realtime 音訊橋接時序、即時轉錄輸出、麥克風控制、browser state refresh
- **解決問題**：Google Meet 參與者可能靜音、無聲音輸出、麥克風控制無效、session 指令在 CLI 退出後中斷
- **影響**：Google Meet 即時語音功能完整可用，包含正確的權限、音訊與控制 ([#76344](https://github.com/openclaw/openclaw/issues/76344))

### ⭐⭐ `doctor --fix` 持久遷移 ([#76798](https://github.com/openclaw/openclaw/issues/76798), [#76800](https://github.com/openclaw/openclaw/pull/76800))

- **用途**：`doctor --fix` 即使有無關驗證問題（如缺失 plugin）也會提交安全的 legacy 遷移
- **解決問題**：其他驗證問題阻止 `agents.defaults.llm` 等已知 legacy key 的清理
- **影響**：升級後的設定清理更可靠

### ⭐⭐ Gateway 效能優化

- **用途**：延遲載入 plugin/runtime discovery、shutdown hooks、cron、channel-config schema、restart sentinels、maintenance timers，精簡重複的 plugin auto-enable 工作
- **解決問題**：Gateway 啟動與 Control UI 熱路徑載入不必要的模組
- **影響**：啟動更快，資源使用更低

### ⭐⭐ Plugin Install/Update 強化

- **用途**：強化 official plugin 的安裝、解除安裝、更新、onboarding、ClawHub fallback、npm dependency-state 報告與 beta-channel 更新路徑
- **解決問題**：externalized plugins 缺乏與 bundled installs 同等的管理體驗
- **影響**：外部插件安裝行為一致且可靠

### ⭐⭐ WhatsApp Channel/Newsletter 目標 ([#13417](https://github.com/openclaw/openclaw/issues/13417))

- **用途**：支援 WhatsApp Channel/Newsletter `@newsletter` 外送訊息目標，使用 channel session metadata 而非 DM routing
