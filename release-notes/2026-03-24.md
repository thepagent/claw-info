# OpenClaw v2026.3.24 版本發佈說明

[GitHub Release](https://github.com/openclaw/openclaw/releases/tag/v2026.3.24)

---

## 概述

### 功能更新 (Features)
- ⭐⭐⭐ **Gateway/OpenAI 相容性** — 新增 `/v1/models`、`/v1/embeddings`，並支援 `/v1/chat/completions` 與 `/v1/responses` 的明確 model override 轉發，提升 RAG 與第三方客戶端相容性
- ⭐⭐⭐ **Microsoft Teams 全面升級** — 遷移至官方 Teams SDK，支援串流回覆、歡迎卡片、打字指示器、原生 AI 標籤等 AI-agent UX 最佳實踐 ([#51808](https://github.com/openclaw/openclaw/pull/51808))
- ⭐⭐⭐ **Microsoft Teams 訊息編輯與刪除** — 支援已送出訊息的編輯與刪除，含 in-thread fallback ([#49925](https://github.com/openclaw/openclaw/pull/49925))
- ⭐⭐ **Skills 一鍵安裝** — 內建技能新增安裝 recipe，CLI 與 Control UI 可在缺少依賴時自動提示安裝 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐⭐ **Control UI/Skills 狀態篩選** — 新增 All/Ready/Needs Setup/Disabled 分頁，技能卡片改為點擊展開詳情對話框 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐⭐ **Slack 互動回覆強化** — 恢復 rich reply 同等功能，自動渲染 `Options:` 為按鈕/選單 ([#53389](https://github.com/openclaw/openclaw/pull/53389))
- ⭐⭐ **CLI/containers** — 新增 `--container` 與 `OPENCLAW_CONTAINER` 環境變數，可在 Docker/Podman 容器內執行 openclaw 指令 ([#52651](https://github.com/openclaw/openclaw/pull/52651))
- ⭐⭐ **Agents/tools 可視性** — `/tools` 顯示當前 agent 實際可用工具，Control UI 新增「Available Right Now」即時區塊
- ⭐⭐ **Discord 自動執行緒命名** — 新增 `autoThreadName: "generated"` 選項，以 LLM 非同步生成簡潔執行緒標題 ([#43366](https://github.com/openclaw/openclaw/pull/43366))
- ⭐⭐ **Plugins/hooks** — 新增 `before_dispatch` hook，含標準化入站 metadata，並透過正常最終交付路徑路由回覆 ([#50444](https://github.com/openclaw/openclaw/pull/50444))
- ⭐ **Control UI/agents** — Agent workspace 檔案列改為可展開 `<details>`，含 lazy-loaded markdown 預覽 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐ **Control UI/markdown preview** — 重新設計預覽對話框，整合 `@create-markdown/preview` v2，支援 light/dark 主題自動切換 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐ **macOS app/config** — 設定頁面側邊欄改為可折疊樹狀結構，取代原有水平 pill 導覽 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐ **CLI/skills** — 缺少需求的標籤從「missing」改為「needs setup」，並在 `openclaw skills info` 顯示 API key 設定指引 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐ **macOS app/skills** — API key 編輯對話框新增「Get your key」連結與儲存路徑提示 ([#53411](https://github.com/openclaw/openclaw/pull/53411))
- ⭐ **Runtime/install** — Node 22 最低版本降至 `22.14+`，避免 npm 安裝時將 Node 22.14 用戶卡在舊版
- ⭐ **CLI/update** — `openclaw update` 前置檢查目標套件的 `engines.node`，Node 版本不符時提供明確升級提示

### 安全性提升 (Security)
- ⭐⭐⭐ **媒體沙箱 bypass 修復** — 關閉 `mediaUrl`/`fileUrl` 別名繞過漏洞，防止出站工具與訊息動作逃脫媒體根目錄限制 ([#54034](https://github.com/openclaw/openclaw/pull/54034))
- ⭐⭐ **出站媒體/本地檔案存取對齊** — 出站媒體存取與 fs policy 對齊，`workspaceOnly` 關閉時保持正常，嚴格模式下維持沙箱隔離

### 錯誤修復 (Bug Fixes)
- ⭐⭐⭐ **Gateway/restart sentinel** — 重啟後透過 heartbeat 喚醒中斷的 agent session，並保留 Telegram topic/Slack thread 路由 ([#53940](https://github.com/openclaw/openclaw/pull/53940))
- ⭐⭐⭐ **Docker/setup** — 修復全新 Docker 安裝在 gateway 啟動前失敗的問題，透過 `openclaw-gateway` 路由設定寫入 ([#53385](https://github.com/openclaw/openclaw/pull/53385))
- ⭐⭐⭐ **Gateway/channels** — 頻道啟動改為循序執行並隔離各頻道失敗，單一頻道錯誤不再阻塞後續頻道 ([#54215](https://github.com/openclaw/openclaw/pull/54215))
- ⭐⭐ **Embedded runs/secrets** — 未解析的 `SecretRef` 設定不再導致 embedded agent 崩潰，改為 fallback 至執行時快照。修復 [#45838](https://github.com/openclaw/openclaw/issues/45838)

- ⭐⭐ **Discord/timeouts** — inbound worker 超時時送出可見的 timeout 回覆 ([#53823](https://github.com/openclaw/openclaw/pull/53823))
- ⭐⭐ **ACP/direct chats** — 最終 TTS 無音訊時仍確保送出 terminal ACP result ([#53692](https://github.com/openclaw/openclaw/pull/53692))
- ⭐ **Telegram/outbound errors** — 保留 403 membership/block/kick 詳情，`bot not a member` 視為永久失敗停止重試 ([#53635](https://github.com/openclaw/openclaw/pull/53635))
- ⭐ **Telegram/photos** — 預先檢查圖片尺寸與長寬比，不符規則時 fallback 為 document 傳送 ([#52545](https://github.com/openclaw/openclaw/pull/52545))
- ⭐ **Slack/runtime defaults** — 修復 Slack DM 回覆開銷、恢復 Codex auto transport，收緊多項 runtime 預設值 ([#53957](https://github.com/openclaw/openclaw/pull/53957))

## 功能更新 (Features) - by Star Rating

### ⭐⭐⭐ Gateway/OpenAI 相容性擴充 ([v2026.3.24](https://github.com/openclaw/openclaw/releases/tag/v2026.3.24))
- **用途**: 新增 `/v1/models` 與 `/v1/embeddings` 端點，並在 `/v1/chat/completions` 與 `/v1/responses` 支援明確 model override 轉發
- **解決問題**: 第三方 OpenAI 相容客戶端（如 RAG 框架、LangChain）無法直接連接 OpenClaw gateway
- **影響**: 任何支援 OpenAI API 的工具皆可直接接入 OpenClaw，大幅降低整合門檻

```text
┌─────────────────┐     POST /v1/chat/completions     ┌──────────────────────┐
│  OpenAI-compat  │  ──────────────────────────────►  │  OpenClaw Gateway    │
│  Client / RAG   │       (with model override)        │                      │
└─────────────────┘                                    │  ┌────────────────┐  │
                                                       │  │ model override │  │
┌─────────────────┐     GET /v1/models                 │  │  forwarding    │  │
│  Client         │  ──────────────────────────────►  │  └────────────────┘  │
└─────────────────┘                                    │         │            │
                                                       │         ▼            │
┌─────────────────┐     POST /v1/embeddings            │  ┌────────────────┐  │
│  Embedding tool │  ──────────────────────────────►  │  │ Agent/Backend  │  │
└─────────────────┘                                    │  └────────────────┘  │
                                                       └──────────────────────┘
```

### ⭐⭐⭐ Microsoft Teams 全面升級至官方 SDK ([#51808](https://github.com/openclaw/openclaw/pull/51808))
- **用途**: 遷移至 Microsoft 官方 Teams SDK，實作 AI-agent UX 最佳實踐：串流 1:1 回覆、歡迎卡片含 prompt starters、feedback/reflection、打字指示器、原生 AI 標籤
- **解決問題**: 舊版整合缺乏 Teams 原生 AI 體驗，用戶體驗與官方 AI bot 標準有落差
- **影響**: Teams 頻道的 AI 互動體驗大幅提升，符合 Microsoft AI-agent UX 規範

```text
                          │  │ 2. 串流回覆 (streaming)   │   │
                          │  │ 3. AI 標籤 (AI label)     │   │

                          │  │  Agent 處理 + 回覆        │   │
                          │  └──────────────────────────┘   │
                          └─────────────────────────────────┘
```

### ⭐⭐⭐ Microsoft Teams 訊息編輯與刪除 ([#49925](https://github.com/openclaw/openclaw/pull/49925))
- **用途**: 支援對已送出的 Teams 訊息進行編輯與刪除，無明確目標時自動 fallback 至 in-thread 操作

## 安全性提升 (Security) - by Star Rating

### ⭐⭐⭐ 媒體沙箱 alias bypass 修復 ([#54034](https://github.com/openclaw/openclaw/pull/54034))
- **用途**: 關閉 `mediaUrl`/`fileUrl` 別名繞過漏洞，確保出站工具與訊息動作無法逃脫媒體根目錄限制
- **解決問題**: 攻擊者可透過 `mediaUrl`/`fileUrl` 別名繞過媒體根目錄沙箱，存取未授權的本地檔案
- **影響**: 修復潛在的本地檔案洩露風險，強化出站媒體安全邊界

```text
┌─────────────────────┐
│  出站工具/訊息動作   │
│  mediaUrl / fileUrl │
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  alias 正規化檢查    │  ◄── 修復前：可繞過
│  (mediaUrl=fileUrl) │
└─────────────────────┘
           │
           ▼
┌─────────────────────┐
│  媒體根目錄邊界檢查  │  ◄── 修復後：統一強制
└─────────────────────┘
           │
      通過 │ 拒絕
           ▼
┌─────────────────────┐
│  允許/拒絕存取       │
└─────────────────────┘
```

### ⭐⭐ 出站媒體存取與 fs policy 對齊 ([v2026.3.24](https://github.com/openclaw/openclaw/releases/tag/v2026.3.24))
- **用途**: 出站媒體存取行為與已設定的 fs policy 對齊，`workspaceOnly` 關閉時保持正常傳送，嚴格模式下維持沙箱隔離
- **解決問題**: 出站媒體存取行為與 fs policy 設定不一致，導致預期外的檔案存取
- **影響**: 媒體存取行為可預測，符合管理員設定的安全策略

## 錯誤修復 (Bug Fixes) - by Star Rating

### ⭐⭐⭐ Gateway/restart sentinel 修復 ([#53940](https://github.com/openclaw/openclaw/pull/53940))
- **用途**: 重啟後透過 heartbeat 喚醒中斷的 agent session，重試一次暫時性交付失敗，並保留 Telegram topic/Slack thread 路由
- **解決問題**: Gateway 重啟後 agent session 無法正確恢復，回覆落在錯誤的 Telegram topic 或 Slack thread
- **影響**: 重啟後對話連續性恢復，訊息路由正確

```text
┌──────────────┐   restart   ┌──────────────────────────────┐
│   Gateway    │ ──────────► │  Restart Handler              │
└──────────────┘             │                               │
                             │  1. 送出 heartbeat            │
                             │  2. 喚醒中斷的 agent session  │
                             │  3. 保留 thread/topic 路由    │
                             │  4. 暫時失敗時重試一次        │
                             └──────────────────────────────┘
                                            │
                                            ▼
                             ┌──────────────────────────────┐
                             │  回覆送達正確 thread/topic    │
                             └──────────────────────────────┘
```

### ⭐⭐⭐ Docker/setup 初始化修復 ([#53385](https://github.com/openclaw/openclaw/pull/53385))
- **用途**: 避免 pre-start `openclaw-cli` shared-network namespace loop，透過 `openclaw-gateway` 路由設定寫入，修復全新 Docker 安裝在 gateway 啟動前失敗的問題
- **解決問題**: 全新 Docker 安裝時，setup 階段在 gateway 啟動前就嘗試寫入設定，導致安裝失敗
- **影響**: Docker 全新安裝流程恢復正常，降低部署門檻

```text
修復前：
┌──────────────┐  setup writes  ┌──────────────┐
│ openclaw-cli │ ─────────────► │  (gateway    │  ← gateway 尚未啟動，失敗
└──────────────┘                │   not up)    │
                                └──────────────┘

修復後：
┌──────────────┐  setup writes  ┌──────────────────┐
│ openclaw-cli │ ─────────────► │ openclaw-gateway  │  ← 透過 gateway 路由
└──────────────┘                └──────────────────┘
                                         │
                                         ▼
                                ┌──────────────────┐
                                │  設定寫入成功     │
                                └──────────────────┘
```

### ⭐⭐⭐ Gateway/channels 啟動隔離 ([#54215](https://github.com/openclaw/openclaw/pull/54215))

- **解決問題**: 一個頻道設定錯誤會導致所有後續頻道無法啟動

```text
修復前：
Channel A ──► 失敗 ──► Channel B、C、D 全部阻塞

修復後：
Channel A ──► 失敗（隔離）
Channel B ──► 正常啟動
Channel C ──► 正常啟動
Channel D ──► 正常啟動
```
